Dynamic Routing for Marine Debris Collection

Reactive planning under ocean currents. Formulated as a Team Orienteering Problem with Moving Targets (TOP‑MT) and solved with a greedy, drift‑aware heuristic with optional retargeting. Explore the methods, the math, and interactive results below.

Start tour ↓ Read the paper

📍 Problem Setting

Plastic patches (hotspots) drift with surface currents. Our objective is to maximize collected value within a time horizon using reactive fleet routing. Each hotspot has a location, a value (proxy for mass), and a simulated drift trajectory.

📐 Mathematical Model

We model the task as a Team Orienteering Problem with Moving Targets. Over a horizon \(T = \{0,1,\dots, \mathcal{T}\}\), a fleet \(K\) collects targets \(N_t\) with values \(\pi_i(t)\) at positions \(p_i(t)\).

\[ \max \sum_{t \in T} \sum_{k \in K} \sum_{i \in N_t} \pi_i(t)\, x_{i,k}(t) \]

Decision variables

  • \(z_{i,k}(t) \in \{0,1\}\): vessel \(k\) targets \(i\) at \(t\)
  • \(x_{i,k}(t) := \mathbf{I}[\lVert q_k(t) - p_i(t)\rVert \le \varepsilon \wedge z_{i,k}(t)=1]\)

Constraints

  • (C1) Unique collection: \(\sum_{t,k} x_{i,k}(t) \le 1\)
  • (C2) One target per vessel & time: \(\sum_{i} z_{i,k}(t) \le 1\)
  • (C3) Vessel dynamics: \( q_k(t{+}1)=q_k(t)+v_k u_k(t)\,\Delta t + c_k(t)\,\Delta t \)
  • (C4) Initial position: \(q_k(0)=q_k^{\text{start}}\)
  • (C5) Non‑overlap pursuit: \(\sum_k z_{i,k}(t)\le 1\)

Scoring (Greedy)

At assignment time, each candidate hotspot receives a score

\[ S_i = \alpha\Big(1-\frac{d_i}{d_{\max}}\Big) + (1{-}\alpha)\Big(\frac{v_i}{v_{\max}}\Big) \]

\(\alpha\in[0,1]\) tunes proximity vs. value. Normalization ensures scale‑invariance across scenarios.

Motion Models

Plastic (Hotspots)

\( p_i(t{+}1)=p_i(t)+c_i(t)\,\Delta t \)

Advected by surface current field at the hotspot location.

Vessels

\( q_k(t{+}1)=q_k(t)+v_k u_k(t)\,\Delta t + c_k(t)\,\Delta t \)

Self‑propulsion in direction of the target plus environmental drift.

Value

Each hotspot has value \(v_i\) (proxy: mass/size/persistence). Once collected, its value becomes 0 and it is removed from \(N_t\).

🧠 Reactive Greedy + Retargeting

  1. Evaluate environment (active hotspots)
  2. Score candidates via \(S_i\) and assign target
  3. Navigate toward target until \(\le \varepsilon\)
  4. Collect, deactivate, and repeat
Extreme strategies underperform: proximity‑only misses high‑value patches; value‑only causes costly detours. Balanced \(\alpha\) wins.

Smart Retargeting

If a near‑path hotspot becomes attractive, a vessel can switch mid‑transit based on an opportunistic score (distance‑biased). This reduces myopia and increases yield per kilometer.

📊 Key Findings

  • Dynamic allocation outperforms static plans by up to ~35% in demanding scenarios.
  • Best performance typically near \(\alpha\approx 0.8\) (proximity‑leaning mix).
  • Retargeting improves both value and distance efficiency across many settings.
