Dynamic Routing for Marine Debris Collection

Optimizing vessel paths in dynamic ocean environments with drifting plastic hotspots

📍 Problem Setting

Marine debris tends to accumulate in oceanic convergence zones. These zones contain floating plastic patches—called hotspots—that vary in size and drift with ocean currents.

Objective: Maximize collected plastic value within a fixed time horizon by choosing which hotspots to visit, and in what order.

⚙️ Mathematical Model

Each hotspot has a value \( v_i \) and a distance \( d_i \) from the vessel. We define a scoring function to balance both:

\[ \text{Score}_i = \alpha \cdot \left(1 - \frac{d_i}{d_{\max}}\right) + (1 - \alpha) \cdot \left(\frac{v_i}{v_{\max}}\right) \]
  • \( \alpha \in [0, 1] \): trade-off parameter
  • \( d_i \): distance to hotspot
  • \( v_i \): value of hotspot

🧠 Greedy Routing Logic

  1. Compute score for all available hotspots
  2. Select the one with the highest score
  3. Travel to and collect it
  4. Re-evaluate and repeat

Changing \( \alpha \) leads to very different behaviors:

  • \( \alpha = 1 \): proximity only → short paths, but low value
  • \( \alpha = 0 \): value only → long detours, high risk
  • Balanced \( \alpha \): optimized trade-off
