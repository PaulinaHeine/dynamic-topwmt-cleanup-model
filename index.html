üìç Problem Setting

Plastic hotspots drift with surface currents. We control one or more vessels that move with self‚Äëpropulsion plus drift. Each hotspot has a position and a value (proxy for mass). Goal: maximize total collected value within the horizon.

How hotspots are modeled

  • Seeding: we seed many floating particles over the surface (initial debris field).
  • Clustering: particles are grouped by proximity into compact clusters ‚Üí each cluster = one hotspot.
  • Value: hotspot value $v_i$ is proportional to its initial cluster size (mass proxy). Once collected, $v_i\to 0$ and the hotspot is deactivated.
  • Drift: both vessels and hotspots are advected by the same surface current reader at their locations.

üìê Math (motion, Œ±‚Äëscore, retargeting)

Motion (discrete time):

$\displaystyle q_k(t{+}1) = q_k(t) + v_k\,u_k(t)\,\Delta t + c\big(q_k(t),t\big)\,\Delta t$
$\displaystyle p_i(t{+}1) = p_i(t) + c\big(p_i(t),t\big)\,\Delta t$

Greedy score (normalized):

$\displaystyle S_i = \alpha\Big(1 - \tfrac{d_i}{d_{\max}}\Big) + (1-\alpha)\Big(\tfrac{v_i}{v_{\max}}\Big)$

Retargeting: while tracking target $j$, switch to $i$ if $S_i \ge S_j + \delta$ (hysteresis). Optionally enforce a minimal binding time to avoid flip‚Äëflops.

Data

  • Currents: surface current vectors from your OpenDrift reader at vessel/patch positions.
  • Values: proportional to initial cluster size (fixed per hotspot in a run).
  • Parameters: $\alpha\in[0,1]$, $\delta\in[0,0.2]$, capture radius $\varepsilon$, time step $\Delta t$.

üß† Greedy Routing Logic

  1. Compute score for all active hotspots
  2. Select best; if already tracking another, apply retargeting rule
  3. Move towards current target with drift
  4. Collect if within radius; remove and repeat
