Dynamic Routing for Marine Debris Collection

Reactive planning under ocean currents. TOP‑MT model, greedy drift‑aware heuristic, and interactive views.

📍 Problem Setting

Plastic patches (hotspots) drift with surface currents. Objective: maximize collected value within a time horizon via reactive fleet routing. Each hotspot has a location, a value (mass proxy), and a drift trajectory.

Problem class
TOP‑MT
Fleet
1–5 vessels (heterog.)
Decision step
Δt (seconds–minutes)
Targets
Drifting hotspots

Hotspots originate from clustered particles. Values are proportional to initial cluster size. On collection, value → 0 and the hotspot is deactivated.

📐 Mathematical Model

We model the task as a Team Orienteering Problem with Moving Targets. Over a horizon \(T=\{0,1,\dots,\mathcal{T}\}\), a fleet \(K\) collects targets \(N_t\) with values \(\pi_i(t)\) at positions \(p_i(t)\).

\[ \max \sum_{t\in T}\sum_{k\in K}\sum_{i\in N_t} \pi_i(t)\, x_{i,k}(t) \]

Decision variables

  • \(z_{i,k}(t)\in\{0,1\}\): vessel \(k\) targets \(i\) at \(t\)
  • \(x_{i,k}(t):=\mathbf{I}[\lVert q_k(t)-p_i(t)\rVert\le\varepsilon\wedge z_{i,k}(t)=1]\)

Constraints

  • (C1) Unique collection: \(\sum_{t,k} x_{i,k}(t) \le 1\)
  • (C2) One target per vessel & time: \(\sum_i z_{i,k}(t) \le 1\)
  • (C3) Vessel dynamics: \(q_k(t{+}1)=q_k(t)+v_k u_k(t)\,\Delta t + c_k(t)\,\Delta t\)
  • (C4) Initial position: \(q_k(0)=q_k^{\text{start}}\)
  • (C5) Non‑overlap pursuit: \(\sum_k z_{i,k}(t)\le 1\)

Scoring (Greedy)

\[ S_i=\alpha\Big(1-\tfrac{d_i}{d_{\max}}\Big)+(1-\alpha)\Big(\tfrac{v_i}{v_{\max}}\Big) \]

\(\alpha\in[0,1]\) tunes proximity vs value. Normalization ensures scale‑invariance.

Motion Models

Plastic (Hotspots)

\(p_i(t{+}1)=p_i(t)+c_i(t)\,\Delta t\)

Advected by surface currents at the hotspot position.

Vessels

\(q_k(t{+}1)=q_k(t)+v_k u_k(t)\,\Delta t + c_k(t)\,\Delta t\)

Self‑propulsion towards the assigned target + drift.

Value

Hotspot value \(v_i\) is proportional to initial cluster size; once collected, \(v_i\to 0\).

🧠 Reactive Greedy + Retargeting

  1. Evaluate active hotspots
  2. Score via \(S_i\) and assign a target
  3. Navigate until within \(\varepsilon\)
  4. Collect, deactivate, repeat
Extreme strategies underperform: proximity‑only misses value; value‑only causes long detours. Balanced \(\alpha\) wins.

📊 Key Findings

  • Dynamic allocation outperforms static plans.
  • Best performance often near \(\alpha\approx 0.8\).
  • Retargeting improves value and distance efficiency.
